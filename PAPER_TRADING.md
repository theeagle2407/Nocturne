# Paper-Trading Log — the Exposure Planner, run on real data

`paper_trading_log.csv` is a verifiable record of Nocturne's **Overnight Exposure
Planner** making nightly decisions on real market data. It is generated by
[`paper_trading_log.js`](./paper_trading_log.js) — the rule is in code, not in a
screenshot, and nothing is hand-edited.

## What each row is

One row per name per night, with the columns the judging form asks for:

| Column | Meaning |
|---|---|
| `date` | the night (enter at the prior close, exit at this day's open) |
| `asset` | ticker |
| `risk_state` | `NORMAL`, `ELEVATED` (vol₁₀ > 1.3 × vol₆₀), or `BLACKOUT` (FOMC day) |
| `action` | the Planner's decision: `HOLD`, `TRIM to $X`, or `FLATTEN` |
| `direction` | `LONG_OVERNIGHT` when exposure is held, else `FLAT` |
| `entry_price` / `exit_price` | prior close / next open |
| `quantity` | shares implied by the (possibly trimmed) position |
| `gross_pnl` / `cost` / `net_pnl` | realized overnight move, 0.10% round-trip cost, and net |
| `account_balance` | running book value after that night |

## The rule (faithful to the live app)

Each night, **causally** (using only data available before that night):

1. Estimate each name's overnight 1σ move from the trailing 60 overnight returns,
   and its risk state.
2. Start from a base overnight position of **$10,000** per name.
3. Apply the Planner overlay: **FLATTEN** blackout (FOMC) names; if the portfolio's
   total 1σ overnight dollar risk exceeds the **$1,000** budget, **TRIM** the
   highest-risk names until it fits; otherwise **HOLD**.
4. Realize the trade net of a **0.10%** round-trip cost.

All parameters are at the top of the script and adjustable.

## How to regenerate

```bash
npm install        # if you haven't already
node paper_trading_log.js
```

This writes `paper_trading_log.csv` and prints a summary. (Run it locally — Yahoo's
public API can block cloud-server IPs.)

## Honest reading of the result

This is **not** presented as a profitable strategy — and it should not be expected to
be. Consistent with Nocturne's backtest, the thin overnight premium does **not** survive
a realistic per-night round-trip cost, so the log lands modestly **net-negative** after
costs. That is the point: it is real, timestamped, risk-managed evidence for Nocturne's
core thesis — *you can't trade your way out of overnight exposure, so you manage it.*
The Planner's value here is the risk control (trimming into volatility, flattening on
FOMC nights), not alpha.
